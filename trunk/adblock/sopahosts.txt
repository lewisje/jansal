# Here’s a tip for the do-it-yourself crowd: Go to your computer's Start menu,
# and either go to "run" or just search for "cmd." Open it up, and type in
# "ping [hostname]" (or preferably "nslookup [hostname]") without quotes.
#
# To automatically create or update such a list, temporarily revert to the
# default HOSTS file, without these special lines, and either whip up a shell
# script using ping or nslookup or use CIP for Windows, available at this
# address: http://www.softsea.com/review/CIP.html
#
# To make a large anti-SOPA HOSTS file more manageable, put hostnames sharing
# the same IP address on the same line (Windows recognizes up to 9 per line),
# as I have done for videoweed.com and novamov.com; also keep in mind that you
# need to specifically list each individual hostname, like both google.com and
# www.google.com (no wildcards allowed). In addition, you cannot specify
# particular ports (like :8080 or :4444), protocols (like https: or ftp:), or
# path components (anything after .com/.org/.tv, after the top-level domain).
# Finally, in cases where a hostname has more than one IP, the IP addresses
# must begin on adjacent lines (at least for Solaris); recording multiple IPs
# allows for the same load-balancing that the multiple A records in the DNS
# currently allow.
#
# Once you have the IP for a website, all you really need to do is enter it
# like you would a normal URL and hit enter/press go. Typing in
# "208.85.240.231" should bring you to the front page of AO3, for example, just
# as typing "174.121.194.34/dashboard" should bring you straight to your Tumblr
# dashboard. Since we’re obviously bracing for the worst case scenario which
# would involve you not being able to access the internet regularly, you should
# save this list.
#
# Consider also setting your DNS servers to BlockAid: http://www.blockaid.me/
# BlockAid currently lets you get to domains seized by the DHS but will also
# circumvent SOPA blocks; the first rule in this HOSTS list is for BlockAid's
# special status page, and the nameserver IP addresses to use for your DNS
# are 178.21.23.150 and...well the other one will come up again.
